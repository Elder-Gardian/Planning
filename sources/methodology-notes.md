# WelfareMap AI 방법론 근거 노트

이 문서는 GitHub Pages의 방법론 보고서가 어떤 정책 결정과 근거를 전제로 하는지 기록한다. P0는 **대상 자치구에 신규 경로당 1개를 설치하는 입지 우선순위 문제**다. 모델은 최종 사업지를 자동 확정하지 않고, 법규·예산·필지 조건을 통과한 후보의 정책 효과를 같은 기준으로 계산해 우선순위를 제안한다.

## P0 의사결정 계약

| 항목 | 확정 내용 |
|---|---|
| 공간 단위 | 100m 격자 1개 = GNN 의사결정 노드 1개 |
| 수요 | 격자·시간대·시나리오별 예상 동시 이용자 `d[i,t,s]` |
| 수요 prior | 65세 이상 인구와 취약성은 설명변수이며 수요 그 자체가 아님 |
| 공급 | 시설 유형별 동시수용량 `K[j,r,t]` |
| 신규 시설 | 경로당 1개 |
| 행동 | `FEASIBLE` 후보 격자 하나 선택 |
| 면적·용량 | 필지 규제와 지정 예산을 적용한 사업비 계산기가 내생적으로 산출 |
| 동일 격자 입지 | 기존 시설이 있어도 별도 필지·증축 가능성이 있으면 허용 |
| 분석 경계 | 수요·신규 후보는 대상구, 경로망은 수도권 전체 |
| 배분 원칙 | 미수용 → 접근부담 상위 꼬리 → 사각지대 → 평균비용 순 사전식 최소화 |
| P0 해법 | 모든 가능 후보 exact evaluation이 기준선, GNN은 top-k 검색 가속 |
| 확장 해법 | 다중 시설부터 beam search·rollout·RL을 비교 |

에이전트는 수요, 기존 시설 용량, 교통상태, 후보 타당성을 바꿀 수 없다. 이 값들은 동일한 시나리오와 난수 시드로 설치 전후 평가에 사용한다.

## 시스템 책임 분리

1. 필지·예산 계산기가 필지별 설치 가능성과 신설 가능한 면적·용량·총사업비를 계산한다.
2. 상태 확장 경로 오라클이 도보·대기·승차·환승을 포함한 고령자 일반화 이동비용을 계산한다.
3. 100m 격자 GNN이 수요·공급·접근성·후보지의 공간적 파급을 인코딩한다.
4. 형평성 배분 솔버가 기존·신규 시설의 용량과 이용자격 안에서 수요를 배정한다.
5. exact evaluator가 설치 전후의 미수용·CVaR90·사각지대·비용 개선을 비교한다.

이 구조에서 GNN은 최단경로 계산이나 수요 배분을 대신하지 않는다. P0에서는 exact evaluator가 정답 오라클이며 GNN은 후보 순위를 빠르게 근사한다.

## 수요·공급 정의

수요 보존식은 다음과 같다.

\[
\sum_j x_{ijt}+u_{it}=d_{it}
\]

시설 용량 제약은 다음과 같다.

\[
\sum_i x_{ijt}\le K_{jrt}
\]

- `d[i,t,s]`: 해당 시간대 시설을 이용할 것으로 예상되는 동시 인원
- `x[i,j,t]`: 격자 `i`에서 시설 `j`로 배정된 동시 인원
- `u[i,t]`: 자격·접근 상한·용량 때문에 배정되지 못한 동시 인원
- `K[j,r,t]`: 시설 `j`가 서비스 유형 `r`에 제공할 수 있는 동시수용량

회원 수, 일 방문자, 등록 정원은 실측 동시수용량의 자동 대체값이 아니다. 경로당·노인복지관처럼 서비스가 다른 시설의 용량도 하나로 합산하지 않는다.

## 이중 그래프와 이동비용

GNN에는 100m 물리 격자 그래프를 사용하고, 경로 계산에는 `(location, status, current_service, last_service, transfer_count)` 상태 그래프를 사용한다. 하차 후에도 `last_service`를 유지해 조금 걸은 뒤 재승차하면서 환승 패널티를 숨기는 경로를 차단한다.

초기 일반화비용은 다음과 같다.

\[
G(p)=1.5T_{walk}+1.75T_{wait}+2.0T_{transferwalk}
+\sum_eT^{ride}_e[p_e^{seat}m_e^{seat}+(1-p_e^{seat})m_e^{stand}]
+5N_{transfer}+B(p)
\]

- 보행속도 기본값 0.85m/s, 민감도 0.70/0.85/1.12m/s
- 좌석·비혼잡 차내시간 1.0, 혼잡 좌석 1.3, 입석 1.6
- 좌석확률 기본값 0.5, 민감도 0.2/0.5/0.8
- 고정 환승비용 기본값 5분, 민감도 0/5/11.2분
- 최대 환승 2회
- 규칙적 배차는 승차 시점마다 `headway/2`, 시각표가 있으면 다음 출발까지의 기대 대기

좌석의 편안함은 음의 보상이 아니라 보행·대기·입석보다 작은 **양의 비용**으로 표현한다. 따라서 합리적으로 조금 긴 좌석 경로는 선택될 수 있지만, 순환할수록 비용은 항상 증가한다. 직접 도보 경로는 교통수단 유무와 관계없이 항상 후보에 남긴다.

경로당의 선호 접근기준은 직접도보 10분을 환산한 `G≤15`, 배정 hard limit은 실제시간 30분과 `G≤45`를 함께 적용한다. 실제시간 hard limit은 20/30/40분으로 민감도 분석한다.

## 필지·예산·용량

필지 상태는 `FEASIBLE`, `HARD_EXCLUDE`, `REVIEW_REQUIRED`, `UNKNOWN`으로 나누고 실제 추천 행동에는 `FEASIBLE`만 허용한다. 도로·하천·철도·최소요건·예산 위반은 hard exclusion이며, 공원처럼 사업유형에 따라 달라지는 토지는 검토 상태로 남긴다.

\[
Cost(p,A)=Land+Demolition+c_{build}(A)A+Design+Supervision+Utility+Permit+Contingency
\]

\[
A_p^*=\max A\quad\text{s.t.}\quad Cost(p,A)\le B,\ A\le FAR_pA_{site},\ A_{footprint}\le BCR_pA_{site}
\]

\[
K_{new,p}=\left\lfloor\frac{\eta A_p^*-A_{fixed}}{a_{user,type}}\right\rfloor
\]

`a_user,type`은 법정 밀도가 아니라 비교 가능한 운영시설의 순사용면적/실제 동시수용량 분포에서 보수적으로 추정한다. 노인복지법상 최소 이용정원과 최소 거실·휴게실 면적은 별도의 hard constraint다.

서울시 2024 공공건축물 공사비 가이드의 경로당 신축 공사비는 200㎡ 미만 385.2만원/㎡, 200~400㎡ 374.2만원/㎡, 400㎡ 초과 368.5만원/㎡다. 이 단가는 토지·철거·설계·감리 등을 포함한 총사업비가 아니며 발주시점 지수로 보정한다.

## 형평성 배분과 보상

배분 솔버는 임의 가중합 대신 다음 순서를 사전식으로 최소화한다.

\[
\operatorname{lexmin}(U,\ CVaR_{0.9}(G),\ Z,\ \bar G,\ Cost)
\]

- `U`: 총 미수용 수요
- `CVaR90`: 접근부담이 가장 큰 상위 10%의 평균
- `Z`: 미수용자와 선호 기준 `G>15` 배정자를 합친 사각지대 수요
- `Ḡ`: 배정 이용자의 평균 일반화비용
- `Cost`: 상위 효과가 같을 때 비교하는 사업비

후보 보상은 설치 후 절대값이 아니라 동일한 기준선 대비 개선량 `R(a)=J_baseline−J_after(a)`으로 계산한다. 하위 목적이 미수용 증가를 상쇄하지 못하도록 사전식 비교 또는 상위 목적의 우선순위를 보장하는 bounded scalar를 사용한다.

## Reward Hacking 불변조건

| 실패 가능성 | 구조적 방어 |
|---|---|
| 가까운 곳도 교통수단 탑승 | 직접 도보 경로 항상 포함, 탑승 자체 보상 금지 |
| 앉아서 오래 순환 | 모든 시간비용 양수, loop·dominance 제거 |
| 환승 숨김 | 하차 뒤에도 `last_service` 유지 |
| 배차 긴 노선 선호 | 최초·환승 승차 때 실제 대기 또는 `headway/2` 부과 |
| 운행하지 않는 노선 이용 | 비용 0이 아니라 비활성 간선 마스크 |
| 이용률 균등화를 위한 먼 배정 | 이용률 분산을 보상에서 제외 |
| 수용량 초과 | 배분 솔버 hard constraint |
| 미도달 수요 삭제 | 항상 `u_i`에 보존, 기준선 전체 수요로 분모 고정 |
| 다른 시설 유형의 가상 공급 | 유형별 용량과 자격 분리 |
| 불법·결측 부지 선택 | `FEASIBLE` 후보만 행동 허용 |
| 행정경계로 지름길 또는 단절 | 대상구 마스크와 수도권 경로망 분리 |

## 검증 계약

- 모든 `FEASIBLE` 후보 exact evaluation과 GNN top-k 결과 비교
- top-1/top-k recall, regret, 추론시간 보고
- 미수용, CVaR90, 사각지대, 실제시간·일반화비용 분포, 총사업비 보고
- 보행속도·수요·좌석확률·환승비용·공사비 민감도 분석
- 직접도보, 환승, 경계 밖 경로, 용량·필지 hard constraint에 대한 단위 테스트
- 추천 후보와 차선 후보의 필지·경로·수혜 격자·탈락 사유 설명

## Pages 시각화 계약

- 핵심 메시지: 정확 평가 가능한 단일시설 P0를 기준선으로 고정하고 GNN은 검색을 가속한다.
- 전체 흐름: 데이터 → 필지·예산 → 경로 오라클 → 격자 GNN → 형평성 배분 → exact 평가
- 숫자 성능처럼 보이는 임의 막대점수는 사용하지 않는다.
- `확정`, `외부 입력`, `연구기반 초기값`, `현장검증 필요`를 문장이나 상태표로 구분한다.
- “AI가 최종 부지를 결정”이 아니라 “검토 가능한 후보의 정책효과와 우선순위를 제안”으로 표현한다.

## 사업별로 입력해야 할 값

- 총예산과 토지매입비 포함 여부
- 신축·매입·임차·증축 중 허용 사업유형
- 시설별 실제 순사용면적·동시수용량·운영시간·이용자격
- 시간대별 예상 이용자와 불확실성 범위
- 공원 등 조건부 설치지역의 사업별 허용 규칙
- 저상버스·역사 편의시설·혼잡·좌석확률의 관측 수준

이 값들은 설계의 미해결 질문이 아니라 실제 사업 시나리오별 외부 입력과 현장검증 항목이다.
