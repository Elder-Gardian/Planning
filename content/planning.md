# WelfareMap AI

## 사업 기획서

### AI 기반 노인 복지 사각지대 탐지 및 신규 경로당 입지 추천

---

## 1. 프로젝트 개요

### 프로젝트명

WelfareMap AI

### 한 줄 소개

WelfareMap AI는 **100m × 100m 격자별 시간대별 예상 동시 이용자 수**, 기존 노인복지시설의 유형별 동시수용량, 고령자 관점의 실제 이동비용, 필지·예산·법규 제약을 함께 분석하여 복지 사각지대를 찾고, 한정된 예산으로 건설 가능한 신규 경로당의 최적 입지를 추천하는 공간 의사결정 지원 시스템이다.

### 프로젝트 목표

이 프로젝트는 지방자치단체와 공공 복지기관이 다음 질문에 데이터로 답하도록 지원하는 것을 목표로 한다.

> 어느 지역의 예상 이용자가 현재 시설에 배정되지 못하거나 과도한 이동 부담을 지고 있으며, 정해진 예산으로 어느 격자에 경로당을 신설해야 그 불균형을 가장 크게 줄일 수 있는가?

WelfareMap AI는 단순히 시설 반경이나 시설 개수를 세지 않는다. 다음 요소를 동시에 반영한다.

- 시간대별 예상 동시 이용 수요
- 시설 유형별 동시수용량과 잔여용량
- 도보·버스·지하철을 결합한 실제 이동 경로
- 노선별 정차순서, 방향, 배차간격, 구간 운행시간
- 환승·대기·보행·계단·경사·승하차 부담
- 좌석에 앉아 이동하는 차내시간의 상대적 편안함
- 시설별 이용자격과 운영시간
- 필지별 토지이용, 건폐율·용적률, 토지가격과 건축비
- 총예산과 법적 최소 시설기준
- 평균 효율뿐 아니라 최악 접근계층을 우선하는 형평성

## 2. P0 범위와 확정 가정

P0는 문제를 검증 가능한 최소 단위로 제한한다.

| 항목 | P0 확정안 |
|---|---|
| 대상 시설 | 도시지역 **경로당** |
| 신규 시설 수 | 1개 |
| 공간 단위 | 100m × 100m 격자 |
| 수요 | 격자·시간대별 **예상 동시 이용자 수** |
| 수요 변경 | 입력 시나리오에서는 변경 가능하나, 에이전트는 에피소드 중 수요를 변경할 수 없음 |
| 기존 공급 | 격자 내 동일 시설유형의 동시수용량 합계 |
| 에이전트 행동 | 신규 시설을 설치할 후보 격자 1개 선택 |
| 신규 시설 규모 | 행동이 아니라, 선택된 격자의 필지·예산 solver가 최대 건축 가능 면적과 수용량을 파생 |
| 총예산 | hard constraint |
| 중복 입지 | 기존 경로당이 있는 격자도 선택 가능. 단, 별도 필지·건물·증축안의 실현 가능성이 있어야 함 |
| 수요·후보 범위 | 분석 대상 자치구 내부 |
| 경로 탐색 범위 | 수도권 전체 교통망. 행정경계 밖으로 나갔다가 돌아오는 경로도 허용 |
| 접근 목표 | 경로당까지 실제 이동시간 10분 이내를 정책 목표로 평가 |
| 배정 상한 | 실제 이동시간 30분을 기본 hard limit으로 사용하고 20·30·40분 민감도 분석 수행 |
| 배분 원칙 | 미수용 수요와 접근성 최악 집단을 우선하는 형평성 사전식 최적화 |
| 기준해 | 모든 설치 가능 후보를 정확 평가하는 exact enumeration |

수요는 단순 65세 이상 인구가 아니라, 시설을 이용하려는 **예상 동시 이용자**다. 100m 격자 노인 인구는 수요를 산출하는 설명변수 또는 사전분포로만 사용한다. 수요 입력은 다음 텐서로 관리한다.

$$
d_{i,t,s}
$$

- $i$: 수요 격자
- $t$: 시간대 또는 계획 시나리오
- $s$: 서비스·시설 유형

예를 들어 평일 오전과 오후의 경로당 예상 동시 이용자가 다르면 서로 다른 $t$로 입력한다. 에이전트가 낮은 수요 시나리오를 선택하거나 수요 자체를 축소해 보상을 높일 수 없도록, 한 에피소드의 수요와 평가 시나리오는 환경이 고정한다.

### 시설 유형별 용량 분리

경로당, 노인복지관, 노인교실은 서비스 범위와 이용자격이 다르므로 모든 용량을 하나로 합치지 않는다.

$$
K_{j,t,s}
$$

시설 $j$의 잔여용량은 시간대 $t$와 서비스 유형 $s$별로 관리한다. 신규 경로당 수요에는 경로당 또는 실제로 대체 가능한 서비스의 용량만 사용할 수 있다. 향후 여러 시설 유형을 함께 다룰 경우 다품목 수요배분 문제로 확장한다.

## 3. 추진 배경과 문제 정의

현재 많은 복지시설 계획은 행정동 단위 통계, 직선거리 또는 일정 반경 안의 시설 개수에 의존한다. 그러나 시설이 가까이 있다는 사실이 곧 이용 가능함을 뜻하지 않는다.

같은 행정동에서도 다음과 같은 이유로 실제 접근성은 크게 달라진다.

- 인근 시설의 동시수용량이 이미 소진됨
- 대로, 철도, 하천, 급경사로 보행 경로가 단절됨
- 정류장은 가까우나 배차간격이 길거나 환승이 필요함
- 역에 엘리베이터가 없거나 환승 과정에 긴 수직 이동이 존재함
- 가까운 시설이 다른 자치구 주민에게 개방되지 않음
- 행정경계 밖 교통망을 잘라 실제 최단경로가 사라짐

반대로 지도상으로 다소 떨어져 있어도 직행 버스나 지하철로 편하게 도달할 수 있고, 좌석을 이용할 가능성이 높다면 고령자에게 더 나은 경로일 수 있다.

WelfareMap AI는 다음 질문에 답한다.

1. 어떤 격자에서 예상 이용자가 시설에 배정되지 못하는가?
2. 어느 격자의 이용자가 10분 접근 목표를 넘는가?
3. 용량 부족, 교통 단절, 환승 부담 중 무엇이 사각지대의 주원인인가?
4. 예산과 토지 제약 아래 신규 경로당을 어느 격자에 설치해야 하는가?
5. 선택한 위치에서 실제로 건설 가능한 면적과 동시수용량은 얼마인가?
6. 평균 접근성이 아니라 접근성이 가장 나쁜 집단까지 얼마나 개선되는가?
7. 추천 결과가 이동수단 탑승이나 수요 삭제 같은 보상 해킹의 결과가 아닌가?

## 4. 핵심 아키텍처

WelfareMap AI는 **100m 격자 의사결정 그래프**, **상태 확장 교통 경로 그래프**, **필지·예산 solver**, **형평성 우선 수요배분 알고리즘**을 결합한다.

```text
공공데이터 및 프로젝트 입력
        │
        ├── 수도권 상태 확장 교통 그래프
        │       └── 격자-시설 간 고령자 일반화 이동비용 사전 계산
        │
        ├── 대상구 100m 격자 의사결정 그래프
        │       └── 수요·공급·교통·취약성·후보지 특징
        │
        └── 필지·토지이용·지가·공사비 데이터
                └── 후보별 건축 가능 면적·용량 계산

100m 격자 그래프 → GNN Encoder → 후보 점수
                                  ↓
                            신규 격자 1개 선택
                                  ↓
                      필지·예산 solver로 면적·용량 파생
                                  ↓
                        형평성 우선 수요 재배분
                                  ↓
                    설치 전후 사각지대 개선량 평가
```

### 역할 분담

- **GNN**: 수요·공급·교통·공간 맥락을 압축하고 유망 후보를 순위화한다.
- **경로 오라클**: 실제 노선, 방향, 배차, 환승과 보행을 반영한 이동비용을 계산한다.
- **필지·예산 solver**: 후보 위치에서 예산으로 건설 가능한 면적과 동시수용량을 결정한다.
- **수요배분 solver**: 용량과 접근 상한 아래 예상 이용자를 시설에 배정한다.
- **정확 평가기**: 설치 전후의 미수용·형평성·사각지대·평균비용 개선량을 산출한다.

이 구조에서는 GNN이 물리적으로 불가능한 면적이나 임의의 수용량을 만들어내지 않는다. 기계학습은 입지 후보의 우선순위를 학습하고, 실현 가능성과 정책 효과는 명시적 solver가 검증한다.

## 5. 그래프 입력 설계

### 5.1 의사결정 그래프: 100m 격자 노드

각 100m 격자는 하나의 GNN 노드다. 대상 자치구 격자와 경계 인접 맥락 격자를 포함하되, 수요와 후보지는 각각 별도의 마스크로 제어한다.

#### 노드 특징

| 구분 | 주요 특징 |
|---|---|
| 수요 | 시간대별 예상 동시 이용자, 65·75·85세 이상 인구, 수요 불확실성 범위 |
| 취약성 | 독거, 저소득, 장애·이동제약 인구 비율 등 확보 가능한 지표 |
| 공급 | 시설 유형별 개수·동시수용량·잔여용량, 운영시간, 이용자격 |
| 현재 접근성 | 가장 가까운 적격 시설 비용, 10·15·30분 내 공급량, 도달 가능한 시설 수, 미배정률 |
| 교통 | 정류장·역 수, 노선 수, 방향별 배차간격, 저상버스·엘리베이터 접근성 |
| 보행환경 | 경사, 계단, 횡단시설, 보행로 단절, 하천·철도·대형도로 장벽 |
| 후보지 | 필지 상태, 가능한 필지 수, 공시지가, 건폐율·용적률, 최대 연면적, 예상 용량·비용 |
| 공간 맥락 | 좌표 positional encoding, 토지이용, 대상구 여부, 경계까지 거리 |
| 기존 시설 | 동일 격자 내 기존 시설 존재 여부와 동시입지 여부 |
| 순차 확장 | 남은 예산, 이미 추가된 공급, 남은 시설 수 |

모델 입력에 설치 후 결과인 미수용량이나 신설 용량을 미리 넣지 않는다. 후보별 파생 가능 용량은 설치 전 공개된 필지·가격·법규만으로 계산한 값이어야 한다.

#### 격자 그래프 간선

- `walk`: 실제 보행로로 연결된 인접 격자
- `spatial_context`: 인접 지역의 수요·토지 특성을 전달하는 공간 문맥 간선
- `accessibility`: 경로 오라클이 계산한 도달 가능 관계 또는 요약 특징
- `barrier`: 하천·철도·간선도로 등 공간적 장벽의 관계

단순 8방향 인접만으로 이동을 계산하지 않는다. 실제 이동비용은 별도의 경로 그래프가 담당한다.

### 5.2 경로 그래프: 노선·방향·탑승상태 확장

격자만 노드로 사용하면 동일 정류장의 여러 노선 사이를 환승 없이 이동하거나, 하차 후 한 격자를 걸은 뒤 재탑승해 환승 패널티를 피할 수 있다. 이를 막기 위해 경로 상태를 다음처럼 확장한다.

$$
(location, stop, route, direction, sequence, onboard, previous\_route)
$$

주요 간선은 다음과 같다.

- 격자에서 정류장·역까지의 접근 보행
- 같은 노선·방향의 다음 정류장 또는 역으로 이동
- 최초 승차
- 하차
- 정류장·역 내부 및 정류장 간 환승 보행
- 목적 시설까지의 최종 보행

대기시간은 모든 구간에 반복해서 더하지 않고, 최초 승차와 환승 후 재승차 시점에만 기본적으로 `배차간격 ÷ 2`를 적용한다. 시간표를 사용할 수 있으면 시간의존 shortest path로 실제 대기를 계산한다.

### 5.3 행정경계와 경로 탐색 범위 분리

다음 마스크를 서로 분리한다.

- $M^{demand}_i$: 대상구 수요로 평가할 격자
- $M^{candidate}_i$: 신규 시설 후보가 될 수 있는 대상구 격자
- $E_{ij}$: 수요 $i$가 시설 $j$를 이용할 자격이 있는지 나타내는 eligibility matrix

교통 그래프는 수도권 전체를 유지한다. 예를 들어 광운대역에서 석계역까지 이동한 뒤 6호선을 타는 경로는 중간에 대상구 경계를 벗어나더라도 허용한다. 계산량을 줄일 때도 행정경계로 자르지 않고, 30분 상한과 안전 여유시간 안에서 도달 가능한 네트워크만 탐색한다.

대상구 밖 시설은 가까우며 이용자격이 있으면 배정 후보가 될 수 있다. 반대로 자치구 주민 전용 시설은 $E_{ij}=0$으로 제한한다. 노원구에서 구로구처럼 현실적으로 먼 배정은 행정구역 hard ban이 아니라 30분 이동 상한과 일반화 이동비용으로 자연스럽게 배제한다.

## 6. 고령자 일반화 이동비용

노인은 도보·대기·환승을 차내 좌석 이동보다 더 부담스럽게 느낄 수 있다. 이를 반영하되 교통수단 탑승 자체에 보상을 주지는 않는다.

경로 $p$의 일반화비용은 다음과 같이 정의한다.

$$
G(p)=\beta_wT_w+\beta_qT_q+\beta_xT_x
+\sum_e T^{ride}_e\left[P^{seat}_e\beta_s+(1-P^{seat}_e)\beta_{st}\right]
+\kappa N_{transfer}+P_{barrier}
$$

- $T_w$: 일반 접근·도착 보행시간
- $T_q$: 대기시간
- $T_x$: 환승 보행·역내 수직이동시간
- $P^{seat}_e$: 구간별 좌석 이용 확률
- $N_{transfer}$: 환승 횟수
- $P_{barrier}$: 경사, 계단, 승하차, 엘리베이터 부재 등의 추가 비용

### 초기 파라미터

| 변수 | P0 기본값 | 민감도 또는 보정 방향 |
|---|---:|---|
| 지속 보행속도 | 0.85m/s | 0.70 / 0.85 / 1.12m/s |
| 일반 보행 가중치 $\beta_w$ | 1.5 | 1.5~2.0 |
| 대기 가중치 $\beta_q$ | 1.75 | 1.75~2.25 |
| 환승보행 가중치 $\beta_x$ | 2.0 | 1.9~2.1 |
| 좌석 차내시간 $\beta_s$ | 1.0 | 항상 양수인 하한으로 유지 |
| 입석 차내시간 $\beta_{st}$ | 1.6 | 1.4~2.4 |
| 고정 환승 패널티 $\kappa$ | 5분 상당 | 0 / 5 / 11분 상당 |
| 최대 환승 | 2회 기본 | 1·2회 민감도, 비용과 시간 상한 병행 |

이 값은 확정된 보편 상수가 아니라 초기 prior다. 서울 고령자 설문과 실제 이용기록 또는 별도의 stated-preference 조사로 교정한다. 고령자 보행 연구에서 평균 약 1.12m/s와 보수적 계획값 약 0.85m/s가 제시된 점을 초기 속도 시나리오에 반영한다. [고령자 보행속도 연구자료](https://idea.seoul.go.kr/upload/suggest/20251127/64D4851EB32848C39BBDFE5FC3006D36.pdf)

2022년 서울 노인실태 자료의 자치구 재분석에서는 주 이동수단이 버스·지하철·도보에 고르게 분포하고, 계단·경사, 승하차, 앉을 곳 부족이 주요 불편으로 나타났다. 따라서 직접도보를 반드시 유지하면서 승하차·수직이동·좌석 가능성을 별도 비용으로 모델링한다. [서울시 노인실태조사 원자료 안내](https://kossda.snu.ac.kr/handle/20.500.12236/30286), [자치구 재분석 보고서](https://council.jungnang.go.kr/kr/bbs/download.do?bbs_id=active&uid=ABC300ABF0C990005BE090C1191909EF)

### 접근성 기준

P0는 경로당의 생활권 목표와 절대 배정 가능 범위를 분리한다.

- 정책 목표: **실제 이동시간 10분 이내**
- 도보 10분의 초기 일반화비용 기준: $G_\tau=15$ equivalent minutes
- 배정 hard limit: **실제 이동시간 30분 이내**이면서 $G\le45$
- 민감도 분석: 실제 이동시간 상한 20·30·40분
- 결과 보고: 10·15·30분 접근권과 일반화비용을 함께 표시

10분은 경로당의 지역밀착형 생활권 목표로 사용하고, 30분은 수요배정 자체를 허용할 절대 기본 상한이다. 10분을 넘었지만 30분 안에서 배정된 수요는 수용은 되었더라도 접근성 사각지대 지표에 포함한다. 접근 기준은 [생활SOC 국가최저기준 연구](https://www.codil.or.kr/filebank/original/RK/OTKCRK230572/OTKCRK230572.pdf)와 서울 노인의 실제 시설 이동시간을 근거로 하되, 대상구 관측자료로 재검증한다.

## 7. 신규 시설의 면적·용량·비용 결정

### 7.1 행동은 위치 하나로 제한

P0 에이전트의 행동은 후보 격자 $a$ 하나를 고르는 것이다.

$$
a\in\{i\mid M^{candidate}_i=1\}
$$

에이전트가 면적이나 수용량까지 직접 출력하면 저렴한 위치에 비현실적으로 큰 시설을 만드는 보상 해킹이 가능하다. 따라서 후보 격자 안의 필지와 예산을 입력받은 deterministic solver가 건설 가능한 최대 연면적과 동시수용량을 계산한다.

### 7.2 필지·예산 solver

필지 $p$와 건축 연면적 $A$의 총사업비는 다음 항목을 포함한다.

$$
C_p(A)=C^{land}_p+C^{demolition}_p+c^{build}_p(A)A
+C^{design}_p+C^{supervision}_p+C^{equipment}_p+C^{utility}_p
+C^{permit}_p+C^{contingency}_p
$$

solver는 다음 제약 아래 용량을 최대화한다.

$$
\max_{p,A} K^{new}_p(A)
$$

subject to:

$$
C_p(A)\le B
$$

$$
A\le FAR_p\cdot A^{site}_p
$$

$$
A^{footprint}\le BCR_p\cdot A^{site}_p
$$

여기에 접도, 주차, 조경, 이격거리, 높이, 지구단위계획, 최소 시설기준을 함께 적용한다.

후보별 사용 가능 프로그램 면적과 신규 동시수용량은 다음처럼 계산한다.

$$
A^{program}_p=\eta_pA^*_p-A^{fixed}_p
$$

$$
K^{new}_p=\left\lfloor\rho_{type}\cdot A^{program}_p\right\rfloor
$$

- $\eta_p$: 연면적 중 실제 프로그램에 사용할 수 있는 비율
- $A^{fixed}_p$: 화장실, 복도, 설비 등 고정 공용면적
- $\rho_{type}$: 시설 유형별 순사용면적 ㎡당 예상 동시수용인원

$\rho_{type}$는 법정 최소면적을 수용밀도로 오해해 정하지 않는다. 실제 운영 중인 비교 가능한 시설의 **순사용면적과 동시수용량**으로 추정하고, 이상치를 제거한 뒤 보수적 분위수를 사용한다. 회원수나 하루 누적 이용자 수는 동시수용량과 동일하지 않으므로 대체값으로 바로 쓰지 않는다.

### 7.3 법적 최소기준

도시지역 경로당은 이용정원 20명 이상, 거실 또는 휴게실 20㎡ 이상의 시설기준을 충족해야 한다. 이 두 최소기준이 1㎡당 1명을 의미하는 것은 아니다. [노인복지법 시행규칙](https://www.law.go.kr/LSW/lsInfoP.do?lsiSeq=282119)과 [노인여가복지시설 시설기준 별표 7](https://www.law.go.kr/flDownload.do?flNm=%5B%EB%B3%84%ED%91%9C+7%5D+%EB%85%B8%EC%9D%B8%EC%97%AC%EA%B0%80%EB%B3%B5%EC%A7%80%EC%8B%9C%EC%84%A4%EC%9D%98+%EC%8B%9C%EC%84%A4%EA%B8%B0%EC%A4%80+%EB%B0%8F+%EC%A7%81%EC%9B%90%EB%B0%B0%EC%B9%98%EA%B8%B0%EC%A4%80%28%EC%A0%9C26%EC%A1%B0%EC%A0%9C1%ED%95%AD+%EA%B4%80%EB%A0%A8%29%0A&flSeq=41476653)을 최종 후보 검증에 적용한다.

### 7.5 동일 격자 중복 입지

기존 경로당이 있는 격자도 자동 제외하지 않는다. 100m 격자로 압축하면 서로 다른 필지나 건물이 같은 노드에 포함될 수 있기 때문이다.

- 별도 건축 가능한 필지가 있으면 신규 건립 허용
- 기존 건물 증축안이 있으면 확충 시나리오로 구분
- 설치 후 같은 유형의 노드 용량에 $K^{new}$를 합산
- `기존 시설 존재`, `동시입지`, `기존 공급 집중도`는 GNN 특징으로 유지

중복 입지는 공급 과잉을 만들 수 있으므로 배분 결과와 형평성 지표에서 자동으로 불리해질 수 있지만, 규칙만으로 사전 금지하지 않는다.

## 8. 필지 설치 가능성 마스크

설치 가능성은 격자 외형이 아니라 격자 안의 필지를 평가한 뒤 집계한다. 후보 필지는 네 상태로 분류한다.

| 상태 | 의미 | P0 action mask |
|---|---|---:|
| `FEASIBLE` | 공개데이터상 예산·면적·토지이용·법적 최소요건을 충족 | 1 |
| `HARD_EXCLUDE` | 도로·하천·철도·공원 등 설치 불가 또는 최소면적·예산 불충족 | 0 |
| `REVIEW_REQUIRED` | 도시계획시설 예외, 공원 내 허가, 경관·고도·지구단위계획 등 별도 심의 필요 | 0, 정책담당자 승인 시 별도 시나리오 |
| `UNKNOWN` | 핵심 필지·권리·규제 데이터 결측 | 0, 자료 보완 후 재평가 |

하나의 격자 안에 `FEASIBLE` 필지가 하나라도 있으면 격자 후보를 활성화하고, solver는 그 안에서 예산 대비 최적 필지를 선택한다. `UNKNOWN`을 임의로 설치 가능 처리하면 추천이 현실에서 무효가 되고, 반대로 모두 제외하면 가용지를 과소평가하므로 별도 상태를 유지한다.

필지 검토에는 연속지적도, 토지이용계획, 도시계획시설, 개별공시지가, 건축물대장과 소유구분 자료를 결합한다. [연속지적도형정보 API](https://www.data.go.kr/data/15123894/openapi.do), [국토교통부 토지이용규제정보](https://www.data.go.kr/data/15123973/openapi.do), [서울시 개별공시지가 API](https://data.seoul.go.kr/dataList/catalogView.do?currentPageNo=1&infId=OA-1180&srvType=A)

공개데이터만으로 권리관계, 지하매설물, 정밀 접도, 실제 매입 가능성까지 확정할 수는 없다. 최종 추천은 후보 순위와 `FEASIBLE confidence`를 제공하고, 사업화 전 현장·법무·건축 검토를 거친다.

## 9. 형평성 우선 수요 배분

신규 시설 후보를 평가할 때 가장 가까운 시설로 단순 배정하지 않는다. 가까운 시설의 희소한 용량을 다른 대안이 없는 수요에게 남겨야 전체 불균형을 줄일 수 있기 때문이다.

수요 격자 $i$, 시설 $j$에 대해 다음 변수를 둔다.

- $x_{ijt}$: 시간대 $t$에 수요 $i$에서 시설 $j$로 배정되는 예상 이용자 수
- $u_{it}$: 어떤 적격 시설에도 배정되지 못한 수요

제약식은 다음과 같다.

$$
\sum_jx_{ijt}+u_{it}=d_{it}
$$

$$
\sum_ix_{ijt}\le K_{jt}
$$

$$
x_{ijt}=0 \quad \text{if } E_{ij}=0 \text{ or } T_{ij}>30 \text{ or } G_{ij}>45
$$

### 사전식 목적함수

형평성은 임의 가중합이 아니라 다음 우선순위의 lexicographic optimization으로 구현한다.

1. **총 미수용 수요** $\sum_i u_i$ 최소화
2. **접근부담 상위 10%의 CVaR90** 최소화
3. **10분 접근 목표 밖 사각지대 수요** 최소화
4. **전체 평균 일반화 이동비용** 최소화
5. 효과가 같은 후보 사이에서 **총사업비와 유휴용량** 최소화

접근성 사각지대 비율은 다음처럼 정의한다.

$$
z_i=\frac{u_i+\sum_jx_{ij}\mathbb{1}[T_{ij}>10]}{d_i}
$$

즉, 시설에 배정되었더라도 10분 생활권 목표를 넘으면 사각지대 개선이 완전히 끝난 것으로 평가하지 않는다. 반면 30분 hard limit을 넘는 수요는 배정할 수 없다.

이 배분은 일부 이용자를 단순히 두 번째로 가까운 시설에 보내, 대안이 없는 다른 이용자의 미수용을 막을 수 있다. 이는 오류가 아니라 고령인구 복지 불균형을 줄이기 위한 의도된 형평성 원칙이다.

## 10. 보상 설계와 Reward Hacking 방지

### 10.1 설치 전후 개선량

보상은 절대 성능이 아니라 동일한 수요·교통 시나리오에서 시설 설치 전후의 목적함수 개선량으로 계산한다.

$$
R(a)=J_{baseline}-J_{after(a)}
$$

$J$는 다음 순서의 벡터다.

$$
J=(U,\ CVaR_{90},\ Z_{10},\ \overline{G},\ Cost)
$$

학습 프레임워크가 scalar reward만 지원한다면 각 지표를 고정 범위로 정규화하고, 상위 우선순위의 최소 개선 단위가 하위 항목의 최대 이득보다 항상 크게 만드는 계층적 계수를 사용한다. 가능하면 constrained 또는 lexicographic policy optimization을 사용한다.

### 10.2 필수 방어 규칙

보상 해킹은 보상 가중치 조정만으로 막지 않고 환경 전이와 평가 규칙에서 차단한다.

| 위험 | 방어 규칙 |
|---|---|
| 도보가 더 편한데도 교통수단 탑승 | 직접도보 경로를 항상 후보에 포함하고, 탑승 자체에는 보상을 주지 않음 |
| 좌석 이동을 무한히 선호 | 좌석 차내시간 비용도 항상 양수로 유지하고 이동시간 hard limit 적용 |
| 불필요한 순환 탑승 | loop 없는 경로와 양의 간선비용 사용 |
| 환승 패널티 회피 | 직전 노선·탑승상태를 경로 상태에 보존하고 다른 노선 재탑승을 환승으로 기록 |
| 매 구간 대기시간 중복 | 최초 승차와 환승 재승차 시에만 대기비용 부과 |
| 낮은 수요만 선택 | 에피소드의 수요·시간대·시나리오를 환경이 고정하고 에이전트 입력에서 불변 처리 |
| 미배정 수요 삭제 | 모든 수요를 $x$ 또는 $u$에 보존하고 설치 전 전체 수요를 고정 분모로 사용 |
| 시설 유형 혼합 | 서비스 유형별 $d$와 $K$를 분리하고 eligibility matrix 적용 |
| 비현실적 대형 시설 생성 | 면적·용량을 행동에서 제외하고 필지·예산 solver로만 파생 |
| 불법 부지 선택 | `FEASIBLE` 후보만 action mask에 포함 |
| 이용률 균등화를 위해 먼 곳 배정 | 시설 이용률 분산을 직접 보상하지 않고 개인 접근비용과 미수용을 평가 |
| 행정경계 절단으로 잘못된 경로 생성 | 수도권 전체 라우팅과 대상구 수요·후보 마스크를 분리 |
| 우연한 혼잡·대기 시나리오 이득 | 설치 전후 평가에 동일 시나리오와 난수 seed 사용 |

대중교통은 도보보다 편할 수 있지만 무료가 아니다. `좌석 차내시간 < 입석·보행·대기·환승 시간`의 관계만 반영하고, 선택된 경로는 언제나 양의 일반화비용을 가진다.

## 12. 학습·탐색 전략

### 12.1 단일 시설 P0: exact enumeration 우선

신규 시설 1개를 대상 자치구의 100m 격자 중에서 고르는 문제는 후보 수가 관리 가능한 경우가 많다. P0에서는 강화학습을 바로 최적해로 간주하지 않고 다음 절차로 모든 `FEASIBLE` 후보를 평가한다.

1. 각 후보 격자에서 필지·예산 solver 실행
2. 파생된 면적과 신규 동시수용량을 같은 유형 공급에 추가
3. 전체 수요를 형평성 원칙으로 재배분
4. $J=(U,CVaR_{90},Z_{10},\overline{G},Cost)$ 계산
5. 후보를 사전식 정렬하여 정확 최적 후보와 전체 순위 생성

이 결과는 정책담당자에게 직접 제공할 수 있는 기준해이며, GNN의 학습 라벨과 검증 oracle로도 사용한다.

### 12.2 GNN의 역할: top-k 검색 가속과 일반화

GNN은 정확 solver를 대체하지 않는다.

- 각 후보의 기대 개선량 또는 순위를 예측
- 상위 $k$개 후보 추출
- 상위 후보마다 필지·배분 solver로 정확 재평가
- exact reranking 결과를 최종 추천으로 사용

P0에서는 전 후보 exact enumeration 결과와 GNN top-k 결과를 비교해 `top-k recall`, regret, 계산시간을 측정한다. 후보가 적은 자치구에서는 exact 결과를 그대로 사용하고, 후보·시설 수·시나리오가 커질 때 GNN의 검색 가속 이점이 생긴다.

### 12.3 왜 P0에서 강화학습을 바로 사용하지 않는가

시설 한 개의 정적 위치 선택에는 순차적 상태전이가 거의 없다. 이 경우 exact enumeration이 가능하면 RL은 불필요한 분산과 보상 해킹 위험을 추가한다. P0는 `정확 최적화 + GNN 후보 랭킹`으로 검증하고, 여러 시설을 순차 배치하는 단계에서 정책학습을 도입한다.

### 12.4 여러 시설로 확장

시설 수가 $m>1$이면 한 시설을 설치할 때마다 다음 상태를 갱신한다.

- 남은 예산
- 선택 가능한 필지
- 유형별 공급과 잔여용량
- 재배분된 미수용 수요
- 격자별 접근성과 형평성 지표
- 동일 격자·인접 격자의 공급 집중도

단순히 현재 가장 좋은 후보를 반복하는 greedy 방식은 두 시설의 상호작용을 놓칠 수 있다. 다음 순서로 확장한다.

1. GNN top-k 후보 생성
2. beam search 또는 rollout으로 여러 배치 조합 탐색
3. 각 단계에서 전체 수요 재배분
4. 마지막 조합을 exact solver로 재평가

시설 수를 고정하지 않는다면 `STOP` 행동을 추가하되, 남은 예산을 쓰지 않는 것이 형평성 개선보다 유리해지는지 명시적으로 평가한다. 시설 개수별 결과를 따로 제시하는 방식이 가장 설명 가능하다.

## 15. 차별성과 기대 효과

### 차별성

**인구가 아닌 예상 이용 수요 중심**

격자 노인 인구를 그대로 수혜대상으로 간주하지 않고 시간대별 예상 동시 이용자를 모델링한다.

**행정경계에 갇히지 않는 실제 접근성**

수요·후보 범위와 수도권 경로 탐색 범위를 분리해 경계 밖 환승과 우회 경로를 보존한다.

**시설 유형과 용량을 구분한 배분**

다른 기능의 복지시설 용량을 합산하지 않고 이용자격·운영시간·서비스 유형별로 배정한다.

**현실적으로 파생되는 시설 규모**

모델이 임의 규모를 제안하지 않고 예산, 필지, 토지가격, 공사비, 법규로 실제 가능한 면적과 용량을 계산한다.

**평균보다 불균형을 먼저 개선**

총효율만 극대화하지 않고 미수용 수요와 접근부담 최악 집단을 우선한다.

**정확 최적화로 검증되는 AI**

단일시설 P0의 exact enumeration을 기준해로 삼고, GNN 결과를 top-k exact reranking으로 검증한다.

**보상 해킹을 구조적으로 차단**

수요, 용량, 경로, 후보 마스크를 환경에서 고정하고 교통수단 탑승 자체에는 보상을 주지 않는다.

### 기대 효과

- 복지 사각지대의 원인과 규모를 격자 단위로 정량화
- 동일 예산에서 가장 큰 형평성 개선을 내는 입지 탐색
- 토지·건축 현실성을 반영한 실행 가능한 후보 제시
- 정책담당자가 이동비용과 수요배정 변화를 추적할 수 있는 설명 제공
- 시설 신설 전 접근성·용량·예산 시나리오 비교
- 향후 노인복지관, 무더위쉼터, 보건소 등 다른 서비스로 확장 가능한 구조 확보

## 16. 한계와 추가 명확화 과제

### 수요 추정의 불확실성

예상 동시 이용자는 조사·예약·이용기록·인구특성으로 추정해야 하며, 관측되지 않은 이용 의향을 완전히 알 수 없다. P0에서는 수요를 외부 입력으로 고정하고 하한·기대·상한 시나리오를 함께 평가한다.

### ㎡당 동시수용량의 실증 필요

법령은 최소 정원과 최소 시설기준을 제시하지만 보편적인 ㎡당 동시수용인원을 규정하지 않는다. 순사용면적과 실제 최대 동시 이용자가 결합된 표본을 확보해 시설유형별 $\rho$를 추정해야 한다.

### 시설 이용자격 확인

자치구 밖 시설을 이용할 수 있는지, 회원 등록이나 프로그램별 제한이 있는지에 따라 배정 결과가 달라진다. 시설별 eligibility matrix를 행정자료로 확정해야 한다.

### 좌석확률과 혼잡도

버스·지하철에서 실제로 앉을 확률은 노선·방향·시간대별로 다르다. P0는 혼잡도 또는 0.2·0.5·0.8 좌석확률 시나리오를 사용하고, 향후 승하차·혼잡 관측으로 개선한다.

### 공개 필지정보의 한계

공개데이터만으로 매입 가능성, 소유권 협의, 지하매설물, 실제 설계조건을 확정할 수 없다. 결과는 예비 후보지 추천이며 최종 건립 타당성 조사와 현장 검토가 필요하다.

### 정책 파라미터 검증

10분 목표, 30분 hard limit, 보행·환승 가중치는 근거를 가진 초기값이지만 대상 자치구의 이용자 조사로 검증해야 한다. 시스템은 값을 숨기지 않고 민감도 분석 결과를 함께 제시한다.

## 17. 단계별 추진안

### P0 — 경로당 1개 정확 입지 추천

- 시간대별 예상 동시 수요 입력
- 기존 경로당 유형별 동시수용량 결합
- 수도권 교통 라우팅과 대상구 마스크 분리
- 필지 4상태 마스크와 예산·용량 solver 구현
- 모든 `FEASIBLE` 격자 exact enumeration
- 형평성 사전식 배분과 보상 해킹 테스트

### P1 — GNN 후보 검색

- exact 순위로 GNN 학습
- top-k recall과 regret 측정
- GNN 상위 후보 exact reranking
- 설명 가능한 후보별 영향권과 비용 분해 제공

### P2 — 다중 시설·다중 시나리오

- 시설 설치 후 수요·공급·예산 상태 재귀 갱신
- beam search 또는 rollout 기반 조합 탐색
- 여러 시간대와 수요 불확실성에 대한 robust 평가
- 신설, 증축, 교통개선 대안 비교

### P3 — 다중 서비스 정책지원

- 경로당·노인복지관 등 서비스 유형별 수요와 용량 확장
- 자치구 간 이용자격과 공동 운영 규칙 반영
- 실제 이용실적 기반 수요·좌석확률·이동비용 재보정

## 18. 최종 비전

WelfareMap AI는 시설이 없는 곳을 표시하는 단순 분류 모델이 아니다.

이 시스템은 예상 이용 수요, 실제 교통 접근성, 시설 유형별 동시수용량, 토지·예산 제약과 지역 간 불균형의 상호작용을 설명하고, 정책담당자가 실행 가능한 신규 경로당 입지를 비교하도록 돕는 **AI 기반 공간 의사결정 지원 시스템**이다.

최종 산출물은 다음을 포함한다.

1. 현재의 100m 격자별 복지 사각지대
2. 시간대별 예상 동시 수요와 미수용 수요
3. 10분 접근 목표 밖 수요와 접근부담 상위 집단
4. 예산·법규를 충족하는 후보 필지와 신규 시설 입지
5. 후보별 건설 가능 면적과 파생 동시수용량
6. 설치 전후 수요배정과 이동경로 변화
7. 미수용·형평성·평균 접근성 개선량
8. 비용 구성과 데이터 불확실성
9. exact 기준해와 GNN 추천의 차이
10. 여러 시설·여러 서비스로 확장 가능한 정책 시나리오

핵심 원칙은 명확하다.

> 수요와 현실 제약은 모델이 바꾸지 못하게 고정하고, AI는 실현 가능한 후보 중에서 고령자의 복지 불균형을 가장 크게 줄이는 위치를 찾는다.
