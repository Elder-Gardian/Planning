# P0 시설 데이터·좌표·용량 보정 결과

- 기준일: 2026-07-17
- 대상: 서울시 도시지역 경로당 3,644건
- 결론: 시설 원천과 P0 좌표값 coverage는 확보했으나, 좌표 검증·실제 동시수용량·행별 운영상태가 미완료라 **최종 입지 최적화는 아직 잠정 단계**다.

## 1. P0 통합 설계 정합성

인수된 P0 통합 설계의 핵심 흐름은 현재 설계와 일치한다.

```text
모든 FEASIBLE 후보 exact 평가
→ exact 결과로 GNN 후보 순위학습
→ top-k exact 재평가
→ 형평성 사전식 목적벡터로 최종 순위 확정
```

상태 확장 교통 그래프, 경계 밖 환승 허용, 직접도보 포함, 환승·대기·입석 비용, 탑승 자체에 보상을 주지 않는 reward-hacking 방지도 유지한다.

다음 세 부분은 현재 계약으로 교정했다.

1. 수요는 모델이 바꾸는 고령인구가 아니라 격자·시간대별 **예상 동시 이용자 수**다.
2. P0 행동은 필지가 아니라 **후보 100m 격자 한 곳**이고, 필지·건축안은 격자 내부 solver가 결정한다.
3. 기존 공급의 고정 8명은 폐기하고 실제 동시수용량 또는 명시된 불확실성 시나리오만 사용한다.

세부 판정과 구현 불변식은 [`p0-design-alignment.md`](https://github.com/Elder-Gardian/Planning/blob/main/report/p0-design-alignment.md)에 있다.

## 4. 고정 8명 대체 방법

### 4.1 8명을 제거한 이유

기존 8명은 설치 신고정원, 소방 수용인원, 좌석 수, 피크 동시이용자 중 어느 것에도 연결되지 않은 일괄 대체값이다. 일부 면적값도 경로당 전용 순사용면적이 아니라 건축물 전체 또는 휴리스틱 면적이므로 공급량으로 사용할 수 없다.

따라서 기존 값은 `legacy_*` 감사 필드에만 남기고 모든 레코드에 `legacy_capacity_excluded=true`를 적용했다.

### 4.2 법령을 실제 용량으로 오해하지 않는 방법

현행 노인복지법 시행규칙 별표 7은 도시지역 경로당에 이용정원 20명 이상을 요구한다. 거실 또는 휴게실 20㎡ 이상은 별도의 시설기준이다. 이는 실제 시설의 최대 동시수용량이 20명이라는 뜻도, `1㎡당 1명`이라는 뜻도 아니다. [노인복지법 시행규칙](https://www.law.go.kr/LSW/lsInfoP.do?lsiSeq=282119), [별표 7](https://www.law.go.kr/LSW/flDownload.do?bylClsCd=110201&flSeq=43368529&gubun=)

소방시설법 시행령 제17조·별표 7은 특정소방대상물의 수용인원을 공간 용도별 바닥면적으로 산정하고 복도·계단·화장실을 제외한다. 경로당의 실제 소방 수용인원은 관할기관 문서값을 우선한다. [소방시설 설치 및 관리에 관한 법률 시행령](https://www.law.go.kr/LSW/lsInfoP.do?ancYnChk=0&chrClsCd=010202&efYd=20260301&lsiSeq=279911&urlMode=lsInfoP)

관할기관 값이 없고 `FACILITY_NET_VERIFIED` 면적만 있는 경우 프로젝트는 `floor(순사용면적/3.0㎡)`를 보수적 민감도 필드로 계산한다. 이는 용도별 실별 면적이 없을 때 쓰는 하향 안전값이며 법적으로 확정된 시설 정원이나 운영 공급량이 아니다. 현재 master에는 검증된 시설 전용 순사용면적이 0건이므로 이 값이 실제 공급에 반영된 시설도 0건이다.

### 4.3 canonical 용량 계층

| 우선순위 | 값 | 용도 |
| ---: | --- | --- |
| 1 | 확인된 운영 동시수용량 | 운영 공급량 |
| 2 | `min(설치 신고정원, 확인 가능한 소방 수용인원, 좌석 수)` | 보수적 운영 공급량 |
| 3 | 피크 관측 동시이용자 | 정원 검증·신규 밀도 보정 자료 |
| 4 | 검증된 전용 순사용면적의 안전 민감도 | 상한 검토용, 운영 공급으로 자동 승격 금지 |
| 5 | 도시 경로당 법정 최소 20명 | 결측 민감도 시나리오 전용 |
| 제외 | 고정 8명, 회원수, 일 방문자, 전체 건축물 면적 | 운영 공급 사용 금지 |

현재 데이터에는 1~3순위 값이 한 건도 없다. 따라서 다음 두 시나리오를 동시에 계산한다.

- `STRICT_UNKNOWN`: 운영상태와 용량이 모두 확인되지 않으면 공급 0명
- `LEGAL_NOMINAL`: 운영불가 4곳을 제외한 3,640곳에 법정 최소 20명을 잠정 적용

`LEGAL_NOMINAL`의 총 72,800명은 실제 공급량 추정치가 아니라 **후보 순위 민감도 검사값**이다. 두 시나리오에서 순위가 달라지는 후보는 실제 정원이 확인되기 전 채택하지 않는다.

### 4.4 실제 정원 확보

노인여가복지시설의 설치와 정원 등 변경 사항은 관할 시·군·구의 신고·변경 신고 자료에서 확인하는 것이 가장 직접적이다. [노인복지법 시행규칙 제25조·제26조](https://law.go.kr/LSW/lsLinkCommonInfo.do?chrClsCd=010202&lspttninfSeq=71701)

25개 자치구에 보낼 [`p0-capacity-register-request.csv`](https://github.com/Elder-Gardian/Planning/blob/main/data/facilities/requests/p0-capacity-register-request.csv)를 3,644건 모두 사전 채움했다. 요청 필드는 다음과 같다.

- 기준일, 현재 운영상태와 휴·폐관·이전 여부
- 확인된 운영 동시수용량
- 설치 신고 이용정원과 변경 이력
- 소방 수용인원과 좌석 수
- 시설 전용 순사용면적, 면적 범위와 신뢰도
- 대표 피크 시간대 관측 동시이용자와 운영시간
- 근거 문서 식별자·URL, 검증자, 확인 메모

회신은 `source_record_id`로 병합한다. `VERIFIED_DISTRICT_REGISTER`, `VERIFIED_SITE_VISIT`, `PARTIAL_VERIFIED` 상태만 적용하며 중복 ID, master에 없는 ID, 음수·분수 인원, 기준일·검증자·근거문서 누락, 전체 건축물 면적 입력은 오류로 중단한다. 확인된 운영상태를 `availability_status`로 변환한 뒤 모든 용량 시나리오와 품질 프로파일을 다시 계산한다.

## 6. 모델에 연결하는 안전 규칙

1. 시설 노드 공급은 `capacity_operational` 또는 명시한 시나리오 열만 읽는다.
2. `legacy_capacity_proxy`, 전체 건축물 면적, 회원수, 일 방문자는 공급 feature에서 차단한다.
3. `availability_status=UNAVAILABLE`이면 어떤 시나리오에서도 용량은 0이다.
4. 대상구 수요·후보 mask, 외부 시설 이용자격 mask, 수도권 routing mask를 분리한다.
5. 직접도보 경로를 항상 포함하고 탑승 횟수·승차 여부에는 보상을 주지 않는다.
6. 모든 보행·대기·차내·환승 비용은 양수이며, 차내 편안함은 시간비용 할인으로만 반영한다.
7. 배분은 수요 보전과 용량 상한을 hard constraint로 두고 `미수용 → CVaR90 → 사각지대 → 평균비용 → 사업비`를 사전식으로 최적화한다.
8. `STRICT_UNKNOWN`과 `LEGAL_NOMINAL`의 후보 순위·regret를 함께 공개한다.

## 7. 완료 판정

이번 작업으로 완료된 범위는 공식 원본 고정, 원천 식별자 보존, P0 좌표값 coverage 100%, 18건 근거 보정, 4건 공급 제외, 고정 8명 차단, 법정 최소 시나리오 분리, 자치구 정원 요청표 생성, 검증·병합 importer 구현이다.

아직 완료되지 않은 범위는 실제 정원·피크 동시이용자 확보, 3,640건 운영상태 확인, 3,642건 좌표 검토, 원천부터의 self-contained 재생, 100m 공간조인, 교차원천 동일시설 승인이다. 이 항목들이 해결되기 전 시설 데이터의 상태는 `PROVISIONAL_NOT_READY_FOR_FINAL_OPTIMIZATION`으로 유지한다.
