WelfareMap AI P0 실증 제안서

고령인구 복지 불균형 완화를 위한 신규 경로당 1개 입지 의사결정 지원

문서 상태 조건부 실증 제안 · 데이터 gate 적용

주요 독자 지자체 의사결정자 · 복지·도시계획 부서 · 실증 파트너

01

1. Executive Summary

  • P0 실증은 착수할 수 있지만 최적 입지를 확정할 단계는 아니다. 서울시 경로당 3,644건의 공식 명부와 좌표값은 확보했으나 좌표 근거 검증 완료는 2건이고, 실제 운영 동시수용량은 확인된 값이 0건이다. 데이터 확인과 모델 구축을 하나의 실증 사업으로 묶되 최종 추천은 명시된 gate를 통과한 뒤에만 허용한다.
  • 의사결정 단위는 100m 격자 한 곳이며, GNN은 후보 검색만 가속한다. 모든 설치 가능 후보를 형평성 우선 수요배분으로 정확 평가한 결과를 기준선으로 두고, GNN이 압축한 top-k 후보도 같은 exact solver로 재검증한다.
  • 고령자의 실제 이동 부담을 왜곡하지 않는다. 직접도보를 항상 선택지에 포함하고, 대기·차내·승하차·환승을 모두 양의 비용으로 계산한다. 앉아서 이동하는 편안함은 차내시간의 제한된 할인으로만 반영하고 교통수단 탑승 자체에는 보상하지 않는다.
  • 우선 의사결정은 대상 자치구, 총예산, 현재 시설대장 확보 방식의 확정이다. 이 세 입력이 정해지면 시설·필지·교통·수요 데이터를 잠그고, exact baseline부터 GNN 후보 검색까지 단계적으로 검증할 수 있다.
제안 결론: 현재 성과는 “최적 입지 도출 완료”가 아니라 검증 가능한 P0 실증의 데이터 계약과 안전장치가 마련된 상태다. 자치구 최신 시설대장과 대상구가 확정되면 본 제안의 단계별 gate에 따라 실행한다.

2. 시설 수가 아니라 미수용과 이동부담을 해결한다

기존의 시설 반경·시설 개수 중심 분석은 같은 거리라도 경사, 환승, 배차간격과 좌석 가능성에 따라 달라지는 고령자의 이동부담을 충분히 반영하지 못한다. 시설이 가까워도 정원이 찼다면 다른 시설로 이동해야 하고, 행정경계 밖을 잠시 우회하는 경로가 실제 최단경로일 수도 있다.

3. P0 의사결정 계약

P0는 대상 자치구에 신규 경로당 1개를 배치하는 위치 추천 문제다. 여러 시설의 순차 배치와 강화학습은 P0 검증 이후 확장 범위로 둔다.

항목P0 계약
수요100m 격자·시간대별 예상 동시 이용자 수, 모델 변경 금지
기존 공급시설 유형별 확인 동시수용량 또는 명시된 시나리오 값
행동FEASIBLE 후보 100m 격자 1곳 선택
필지선택 격자 내부에서 결정론적 solver가 실현 가능한 필지·건축안을 선택
신설 규모예산, 토지비, 공사비, 건폐율·용적률, 법정 최소기준으로 결정
이동 범위수도권 전체 교통망에서 경로 탐색, 시설 이용자격은 별도 제한

동일 격자에 기존 경로당이 있어도 별도 필지, 별도 건물 또는 실현 가능한 증축안이 존재하면 후보가 될 수 있다. 다만 도로·하천·공원·도시계획시설과 법규상 불가능한 필지는 HARD_EXCLUDE로 제거한다.

4. 제안 시스템은 두 그래프와 하나의 exact 기준선으로 구성한다

수요·시설·필지·교통 데이터 잠금
→ 모든 FEASIBLE 후보의 exact 수요배분 평가
→ exact 결과로 GNN 후보 순위학습
→ GNN top-k 후보 exact 재평가
→ 형평성 목적벡터와 근거자료를 함께 제출

4.1 100m 의사결정 그래프

각 100m 격자를 하나의 노드로 두고 예상 동시수요, 기존 시설의 유형별 공급, 접근성, 설치 가능성, 토지가격·공사비와 인접 격자 정보를 특징으로 넣는다. 이 그래프는 어디에 설치할지를 표현한다.

4.2 상태 확장 교통 그래프

버스·철도의 노선, 방향, 정차순서, 배차간격과 현재 탑승 노선을 상태로 보존한다. 도보, 승차, 차내 이동, 하차와 환승을 서로 다른 간선으로 구성해 어떻게 이동하는지를 계산한다. 구간 운행시간이 없으면 노선별 보정시간을 사용할 수 있지만, 대기와 환승은 별도 비용으로 남긴다.

4.3 형평성 우선 exact 배분

수요는 시설 정원을 넘지 않도록 배분하며, 수용하지 못한 이용자와 가장 부담이 큰 상위 10%의 이동비용을 평균보다 먼저 줄인다. GNN 점수가 높아도 exact 배분에서 용량·예산·입지 제약을 통과하지 못하면 추천하지 않는다.

6. 고정 8명 대신 근거와 불확실성을 분리한다

기존 시설당 8명은 신고 이용정원, 소방 수용인원, 좌석 수 또는 피크 동시이용자와 연결되지 않아 공급량에서 전부 제외했다. 과거 데이터에 존재하는 회원수, 일 방문자와 전체 건축물 면적도 동시수용량으로 사용하지 않는다.

자치구 회신 전에는 다음 두 민감도 시나리오를 분리해 평가한다.

  • STRICT_UNKNOWN: 운영상태와 용량이 확인되지 않은 시설의 공급을 0명으로 처리
  • LEGAL_NOMINAL: 운영불가 4곳을 제외한 3,640곳에 법정 최소 20명을 적용한 순위 민감도 검사

LEGAL_NOMINAL 합계 72,800명은 서울시 경로당의 실제 총수용량이 아니다. 두 시나리오에서 추천 순위가 바뀌면 실제 정원 확인 전에는 후보를 채택하지 않는다.

신규 시설은 검증된 순사용 프로그램 면적과 관측 기반 피크 동시이용 밀도로 용량을 산정한다. 관측 밀도가 없으면 여러 보수적 밀도 시나리오를 공개하고, 예산으로 법정 최소기준을 충족하지 못하는 필지는 설치 불가로 처리한다.

07

7. 경계 밖 경로는 허용하되 비현실적 시설 배정은 막는다

행정경계는 서로 다른 세 규칙으로 분리한다.

  1. 대상구 수요와 신규 후보는 사업 범위 안으로 제한한다.
  2. 이동 경로는 수도권 전체 교통망에서 탐색해 경계 밖 환승과 우회를 허용한다.
  3. 외부 시설은 이용자격과 고령자 일반화 이동시간 hard limit를 모두 통과할 때만 배정한다.

이 구조는 광운대역에서 석계역으로 이동해 6호선을 타는 유효 경로를 보존하면서, 노원구 수요가 단순히 빈자리가 있다는 이유로 구로구 시설에 배정되는 결과를 막는다.

08

8. Reward Hacking은 보상 조정이 아니라 구조적 불변식으로 차단한다

위험방어 규칙P0 검증 사례
가까운 거리도 억지로 대중교통 이용직접도보를 모든 OD의 후보로 포함, 지배당한 경로 제거도보가 싼 경우 환승 경로가 선택되지 않음
앉아서 간다는 이유로 긴 우회 선택차내 편안함은 제한된 시간비용 할인, 비용은 항상 양수우회시간 증가가 무제한 보상되지 않음
환승 패널티 회피직전 노선·방향·탑승상태를 경로 상태로 유지동일·타 노선 환승비용이 정확히 부과됨
순환 탑승으로 보상 누적탑승 자체 보상 금지, 양의 간선비용과 loop 제거반복 승하차가 비용을 낮추지 않음
수요 축소로 성과 부풀리기수요 입력 불변, 배정량+미수용량=전체 수요모든 시나리오에서 수요 보전
시설 정원 초과용량 상한을 hard constraint로 적용어떤 후보도 시설별 정원을 초과하지 않음

이 항목들은 현재 구현 완료 성과가 아니라 routing·solver 구현 시 반드시 통과해야 하는 회귀 테스트 계약이다.

09

9. 단계별 실증 범위와 산출물

단계핵심 작업주요 산출물완료 gate
A. 사업 입력 잠금대상구·예산·예상 동시수요·정책 파라미터 확정P0 데이터 계약서, 시나리오 정의입력 단위와 기준일 승인
B. 시설·좌표 정합화최신 운영상태·정원 회신, 좌표·100m 공간조인검증 시설 master, 품질 리포트대상구와 이용 가능 인접시설 검증
C. 경로 오라클수도권 상태 확장 그래프와 고령자 비용 계산시간대별 OD 비용·경로 근거경계 밖·도보 우위·환승 golden test 통과
D. 필지·사업규모 solver지적·용도지역·도시계획시설·비용 결합4상태 입지 mask, 격자별 실현 사업안FEASIBLE 후보만 평가
E. exact 기준선모든 후보의 용량 제약 수요배분baseline 순위, 형평성·비용·regret수요 보전·용량 상한·0수요 검사 통과
F. GNN 후보 검색exact 결과로 top-k 순위 모델 학습GNN 모델, Recall@k·속도·regret 비교top-k exact 재평가 후 정책 순위 보존
G. 정책 검토 패키지시나리오와 후보 근거를 의사결정 문서화후보 카드, 지도, 민감도·감사 로그담당부서가 근거를 재현할 수 있음

기간과 인력은 대상구 범위, 교통 데이터 라이선스, 자치구 회신 SLA와 필지자료 제공 범위가 확정된 뒤 산정한다. 행정 회신을 기다리는 동안 경로 오라클과 exact solver를 병렬 구축하되, 미확인 값을 실제값으로 대체하지 않는다.

10. 성공은 모델 점수보다 정책 순위의 안정성으로 판정한다

검증 축핵심 지표판정 원칙
형평성미수용 인원, CVaR90 이동부담, 선호시간 밖 사각지대평균시간 개선보다 먼저 비교
접근성직접도보·대중교통 일반화비용, 접근 가능 수요 비율시간대·경로 근거를 함께 제시
사업성총사업비, 건축 가능 면적, 신규 동시수용량예산·법규 위반 후보 0건
데이터운영상태·용량·좌표 검증률, 기준일결측과 대체값을 별도 열로 공개
GNNexact top-k Recall, NDCG, 정책 regret, 계산시간exact 순위를 얼마나 보존하는지 평가

모델 성능 목표값은 대상구의 후보 수와 exact baseline 분포를 확인한 뒤 설정한다. 사전에 임의 정확도를 약속하지 않고, exact 대비 누락과 정책 regret를 공개한다.

11. 착수에 필요한 결정과 추가 질문

즉시 필요한 세 가지 결정

  1. P0 대상 자치구와 인접 시설의 이용자격 범위를 어디까지로 할 것인가?
  2. 토지비·설계비·공사비를 포함한 총예산과 예비비 규칙은 무엇인가?
  3. 자치구 최신 경로당 대장과 시설별 운영·정원 확인을 누가 어떤 기준일로 제공할 것인가?

모델 입력 전 합의할 항목

  • 예상 동시 이용자 수의 기준 시간대와 추정 방식
  • 고령자가 감당 가능한 최대 일반화 이동시간
  • 좌석 가능성, 경사, 날씨와 환승 패널티의 초기값 및 민감도 범위
  • 동일 격자 중복 입지에서 허용할 신축·증축·복합화 유형
  • 외부 자치구 시설의 실제 이용 가능 여부와 서비스 협약
  • 추천안의 정책 승인 기준과 공개 가능한 설명 수준
